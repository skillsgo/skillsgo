# SkillsGo Support

Choose the channel that matches your request:

- [GitHub Discussions](https://github.com/skillsgo/skillsgo/discussions) for usage questions, troubleshooting, integration advice, and early ideas.
- [Bug report](https://github.com/skillsgo/skillsgo/issues/new?template=bug_report.yml) for reproducible product defects.
- [Feature request](https://github.com/skillsgo/skillsgo/issues/new?template=feature_request.yml) for a concrete user problem and desired outcome.
- [Documentation issue](https://github.com/skillsgo/skillsgo/issues/new?template=documentation.yml) for incorrect, missing, or unclear documentation.
- [Private vulnerability report](https://github.com/skillsgo/skillsgo/security/advisories/new) for security issues.

Community support is provided on a best-effort basis. Include the SkillsGo version, operating system, target Agent, public Repository or Skill coordinate, and sanitized diagnostics when relevant. Never post credentials, tokens, private source code, or other sensitive information.

Discussions may be written in any language. For a non-English opening post, repository automation may add a clearly labeled machine-generated English translation and summary so the conversation remains searchable across the community. The original post remains authoritative, and maintainers may correct translation errors.

Issues track actionable defects and accepted work. Maintainers may convert support requests into Discussions, close duplicates, or request additional information before triage.
