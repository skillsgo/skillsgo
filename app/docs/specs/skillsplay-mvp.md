# Historical SkillsPlay macOS MVP

This document records the first validation scope: a closed macOS client, one Codex target, user-level installations only, an external `skills` CLI, and read-only `skills.sh` discovery APIs.

That scope is no longer an implementation source. It has been superseded by [User Journeys and Information Architecture](../user-routes.md) and [SkillsPlay Personal MVP](../mvp.md), which define the multi-Agent, multi-project, SkillsGo Registry, and bundled CLI product.

The historical decisions remain summarized here so future contributors understand why early Flutter code still contains external CLI detection, Codex-only commands, and search-only discovery states.
